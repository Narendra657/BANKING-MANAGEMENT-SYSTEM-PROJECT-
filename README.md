# BANKING-MANAGEMENT-SYSTEM-PROJECT-
A relational database project built with MySQL simulating core banking operations — including customer management, accounts, transactions, and loans. Covers DDL/DML/DQL, JOINs, aggregate functions, stored procedures, triggers, and CASE expressions. 

This project simulates the backend database of a commercial bank. It handles customer onboarding, savings/current/fixed deposit account creation, deposit and withdrawal transactions with automatic balance updates via triggers, fund transfers between accounts through stored procedures, and loan management for Home, Car, Personal, and Education loans. Queries cover financial reporting using aggregate functions, customer lookup using pattern matching, multi-table data retrieval using JOINs, and dynamic balance categorization using CASE expressions — making it a complete demonstration of MySQL in a real-world finance domain.


Use Cases Of This Projects:

Register and manage bank customers with unique contact details
Open Savings, Current, or Fixed Deposit accounts per customer
Record deposits, withdrawals, and inter-account transfers
Auto-update account balance after every transaction via trigger
Transfer funds between two accounts using a stored procedure
Track Home, Car, Personal, and Education loans with interest rates and tenure
Generate financial summaries — total bank balance, average balance, min/max balance
Filter and search customers by name, city, email pattern, or account type
Categorize accounts as Low, Medium, or High balance using CASE logic
Retrieve cross-table reports linking customers, accounts, and loan data via JOINs
